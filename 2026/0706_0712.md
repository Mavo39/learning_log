```python
# 問題
# 権限は、閲覧のみ、コメントの記入可、編集可の 3 種類で、ホワイトボードを編集するには 3 つとも権限を持っている必要があります。またどの権限も持っていない場合は入室することができません。ユーザーが 3 つの権限をすべて持っているか判別する関数を作成してください。

# ユーザー権限 閲覧のみ、コメント記入可、編集可
PERMISSIONS = ('viewOnly', 'addComment', 'editable')

class User:
    def __init__(self, userId, name, permissions):
        self.userId = userId
        self.name = name
        self.permissions = permissions

    # 受け取った権限を持っているか返す関数を作成してください ← ※1
    def hasPermission(self, permission):

	# 正しい表現
		return permission in self.permission

	# 自分が書いた表現
	# ここでいう「受け取った」は関数の引数として指定した permission がユーザーの権限に含まれているかどうかということ
	# 誤解ポイント: 3つの権限をすべて持っているかをチェックしようとした
		return all(permission in self.permission for permission in PERMISSIONS)
```

- 上記の設計でわざわざ `hasPermission()` というメソッドを作らなくてもいいのでは？と思ったが、このように「権限チェック」という共通の土台を作っておくことで、"編集権限"・"コメント投稿権限"といった個別具体的なチェック機能を量産できる、という拡張性が担保できることがわかった
- また、「権限をチェックする」という意図が明確になり、可読性の向上にも貢献できるとわかった
- 抽象的な判定ルールと具体的なシーンの判定を分離することで、将来の変更に備えているという視点
