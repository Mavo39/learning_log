```python

is_canceled = False
is_repeater = True  # 今回の判定には関係ないノイズ

    if is_paid:
        if not is_shipped:
            print("決済完了・発送待ち")
        else:
            print("発送済み")
    else:
        print("未決済")
else:
    print("キャンセル済み")

# ビット操作
# 状態の定義: どの桁をチェックするか
PAID      = 0b00001  # 1: 決済
SHIPPED   = 0b00010  # 2: 発送
CANCELLED = 0b00100  # 4: キャンセル
REPEATER  = 0b01000  # 8: リピーター

# 現在の状態
# 決済(1) と リピーター(8) がONなので、0b01001 になる

# スポットライト（マスク）の作成: 決済と発送の桁だけをチェック対象にする
# "OR" で繋いでいるのは、チェックしたいフラグを明らかにするため
CHECK_MASK = PAID | SHIPPED  # 0b00011

# 理想の形：決済がON(1)で、発送がOFF(0)の形
TARGET_STATUS = PAID  # 0b00001

# AND演算で型抜きして一発判定
# 0b01001 と 0b00011 をANDすると、無関係なリピーターの桁が消えて 0b00001 になる
if (current_status & CHECK_MASK) == TARGET_STATUS:
    print("決済完了・発送待ち")
else:
    print("それ以外の状態")
```

- このビット操作によって、すべての状態の組み合わせを "0と1の並び" だけで管理できるようになり、1バイト（8ビット）という極小のメモリ領域の中で、最大8つの条件を同時に管理できる。そのため、仮に現在の状態管理は5つで将来的に条件が増えたとしても、1バイト内なら、あと3条件は追加できることになる
- 論理演算子とビット演算子の使い分けの境界線がクリアになった、同じ意味だと勘違いしていたことに気づいた
