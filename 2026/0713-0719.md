- 「短く書くこと ≠ 綺麗に書くこと」可読性やメンテナンス性も考慮し、デバッグしやすいかの方が重要(※1)

```python
# ※1
def NOT(p):
    return not p

def OR(p,q):
    if p == 1 or q == 1:
        return True
    else:
        return False

def AND(p,q):
    if p == 1 and q == 1:
        return True
    else:
        return False

# デバッグしやすい
def circuit1(p,q):
    s = NOT(p)
    t = NOT(q)
    u = AND(s,q)
    v = AND(p,t)
    r = OR(u,v)
    
    return r

# ネスト構造
def circuit1(p,q):
    return OR(AND(NOT(p), q), AND(p, NOT(q)))
```

- マスク処理: データの中で必要な部分だけ抽出し、不要な部分を覆い隠す処理
- ビット演算においては、2進数の特定の桁だけ取り出したり、クリア（0）にしたりする処理に使われるらしい
- `何か & 1`: 元の値をそのまま返す

```python
def NOR(p, q):
    return ~(p | q) & 1
```

- ド・モルガンの法則では、全体にかかっている否定（バー）を個別にばらすとき、真ん中の演算子（and と or）が必ずセットでひっくり返るという性質に対する理解が漏れていることに気づいた
